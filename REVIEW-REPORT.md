## ⚠️ ACTION REQUIRED (before the iOS on-device sync test)

1. **Run `supabase/10-security-hardening.sql`** in the SQL Editor. It fixes:
   - `event.id` was `GENERATED ALWAYS` → every CSV import in sync mode was
     **silently erased** (Postgres rejects explicit ids → connector drops the
     ops → next checkpoint wipes the local rows). One-line `SET GENERATED BY
     DEFAULT` fixes the whole feature.
   - Email-match account linking trusted **unconfirmed** emails — with
     "Confirm email" OFF, anyone could sign up as one of the 11 legacy web
     users' addresses and inherit their tasks. The trigger + `03` backfill now
     require `email_confirmed_at`.
   - Any authenticated user could plant tasks in a *web* user's list via the
     unprotected legacy `user_id` column (column-level grants now exclude it).
   - Anyone could insert a completion row against a *victim's* recurring task
     (guessable sequential id) and permanently block their daily check-off via
     the unique constraint. WITH CHECK now requires ownership.
   - `avatars` bucket had no size/MIME caps = free public file hosting under
     your project domain. Now 1 MB, images only.
2. **Decide: "Confirm email" stays ON, permanently.** It's currently ON (by
   accident, flagged 2026-07-11). The audit reclassified it: with email-match
   account linking in the schema, confirm-email is a **security invariant**,
   not a dev-phase convenience. If you turn it OFF, the `10` fix still guards
   the legacy-task takeover, but new signups with someone else's address
   remain possible. Recommendation: leave ON; the deployed web app already
   receives the confirm links.

## Deferred — needs your judgment (nothing here is broken)

**Design decisions (from the design audit — report-only per your instruction):**
1. **Urgent vs ongoing is color-alone** on cards and calendar dots (your
   non-negotiable rule 1 / WCAG 1.4.1). Overdue has its icon; urgent needs a
   non-color cue too — suggestion: small clock glyph next to the due date, or
   a doubled accent bar. Your call on the shape.
2. **Import sheet shows two filled buttons at once** ("Choose a different
   file" + "Import n events") — rule 3 violation. Suggestion: demote the first
   to a text link once a file is parsed; also its fill is the event blue,
   which doc 03 explicitly names as the generic-AI accent to avoid.
3. **Sheet shell is copy-pasted across 5 routes** (form/styles/import/help/
   event) with ad-hoc 220/280/260ms timings that aren't in doc 05's motion
   vocabulary. Suggestion: extract one `SheetShell`, canonize the tuned values
   as named motion tokens (like the swipe values were), and the reduced-motion
   + drag-to-dismiss behavior lands in all five at once.
4. **Three chip implementations with three geometries** (form 40pt, filter bar
   36pt, settings 40pt). One shared `Chip` in components/ would fix doc 02's
   "don't invent one-off styles" rule; hitSlop covers the a11y minimum today.
5. Smaller spec drifts, worth a doc-hygiene pass more than code changes:
   accent bar 3px vs doc's 2px (pick one, record it); 13pt font sizes (off the
   12/16 scale) on date chips + filter chips; calendar "+n" at 9pt; section
   chevrons swap instead of rotating; sync-status dot colors differ from doc
   02 (update the doc — the code is the newer decision); FAB→sheet
   choreography in doc 05 was never built (trim the doc or build it).
6. **Sign-in/sign-up design pass** (known placeholders): the audit's checklist
   when you get there — token radii/type, doc 02 primary button with in-place
   loading spinner, visible labels, focus states, error text under fields,
   and don't use `statusOngoingAccent` for the sign-up notice (status colors
   are for task state).
7. Styles sheet copy ("New looks for your tasks…") reads marketing-ish vs the
   app's tool voice — one factual line would fit doc 06 better.

**Technical, deliberately not done:**
8. **Session tokens live in plain AsyncStorage** — fine for dev; swap in the
   SecureStore adapter (Supabase's Expo guide) at the dev-build stage.
9. **`components/input-prompt.tsx` uses RN `<Modal>`** — the one sanctioned
   exception (static content only, documented inline). Convert to a
   transparentModal route during the Android pass if you want rule 3 absolute.
10. **Android keyboard double-compensation risk** in the form sheet (manual
    padding + `adjustResize`) — flagged for the Android pass checklist.
11. **Dot-separated CSV dates parse month-first** (`1.2.2026` = Jan 2); dots
    conventionally mean day-first in Europe. Either drop dot support or parse
    day-first — data-shape decision, left alone.
12. **List memoization** (React.memo on cards) — worth doing if typing in
    search ever janks on-device; measure first.
13. **"Today" goes stale across midnight** on Daily/calendar until a
    re-render; an AppState/interval tick would fix it. Cosmetic.
14. **Dropped sync ops are invisible** — the counter now exists
    (`getDroppedOpCount()` in the connector); surfacing it belongs to the
    sync-status indicator when you build it (docs/design/02).
15. `expo-notifications` plugin icon/color for Android (needs a monochrome
    icon asset — bundle with the Android pass).
16. Deleted-section tasks: **event colors on the calendar ring** still use the
    raw CSV color (only *text* got the clamp) — visually intentional, noting
    for completeness.

**Residual known edge (accepted):** if two task mutations are BOTH in flight
offline in REST mode and BOTH fail, the cache can keep one phantom change
until the next successful refetch. Sync mode (the real offline path) doesn't
have this — local writes don't fail. Fixing it fully needs per-entity inverse
rollback; not worth the complexity now.
